# AT&T Face Recognition System - Comprehensive Guide

## Project Overview

This project is a face recognition system that uses the AT&T Face Database (also known as Olivetti Faces) to demonstrate different machine learning approaches to facial recognition. The system implements and compares four different algorithms:

1. **Support Vector Machine (SVM)** - A supervised learning algorithm
2. **Neural Network (MLP)** - A deep learning approach using Multi-Layer Perceptron
3. **Principal Component Analysis (PCA) with SVM** - Dimensionality reduction combined with SVM
4. **K-means Clustering** - An unsupervised learning approach

The project provides two user interfaces:
- A desktop application built with Tkinter
- A web application built with Streamlit

## Core Components

### 1. MLPP.py (Main Face Recognition System)

This is the core module that implements all the machine learning functionality:

- **Data Loading**: Uses scikit-learn's `fetch_olivetti_faces()` to load the AT&T Face Database, which contains 400 grayscale images of 40 individuals (10 images per person).
- **Data Preprocessing**: Splits the data into training and testing sets, normalizes the images.
- **Model Training**: Implements four different models:
  - SVM for classification
  - Neural Network (using MLPClassifier) for deep learning approach
  - PCA for dimensionality reduction combined with SVM
  - K-means for unsupervised clustering
- **Performance Evaluation**: Calculates accuracy, generates confusion matrices, and produces classification reports
- **Visualization**: Creates plots for model comparisons, confusion matrices, eigenfaces, etc.
- **Prediction**: Processes new images and predicts the person's identity using the trained models
- **Model Persistence**: Saves and loads models for future use

### 2. tkinter_app.py (Desktop Application)

The Tkinter application provides a user-friendly desktop interface with multiple tabs:

- **Home Tab**: Displays general information about the system and the dataset
- **Recognition Tab**: Allows users to upload images or use a webcam for face recognition
- **Performance Tab**: Shows model performance metrics and visualizations
- **Dataset Tab**: Enables browsing through the AT&T Face Database by person

### 3. streamlit_app.py (Web Application)

The Streamlit application provides a web-based interface with similar functionality:

- Interactive model performance visualization
- Face recognition from uploaded images
- Dataset exploration capabilities

## Application Workflow

### MLPP.py Workflow

1. **Initialization**: 
   - Loads the AT&T Face Database
   - Splits data into training and testing sets
   - Initializes model storage

2. **Model Training**:
   - `train_svm()`: Trains an SVM model on flattened image data
   - `train_cnn()`: Trains an MLP neural network on the data
   - `train_pca_svm()`: Applies PCA for dimensionality reduction, then trains SVM
   - `train_kmeans()`: Performs clustering and maps clusters to the most common person

3. **Visualization**:
   - `plot_accuracy_comparison()`: Compares model accuracies
   - `plot_cnn_history()`: Shows training history for the neural network
   - `plot_confusion_matrix()`: Creates confusion matrices for each model
   - Visualizes eigenfaces for PCA components

4. **Prediction**:
   - `predict_new_image()`: Takes a new image, preprocesses it, and makes a prediction using the selected model

5. **Model Persistence**:
   - `save_models()`: Saves trained models to disk
   - `load_models()`: Loads models from disk

### Tkinter App Workflow

1. **Initialization**:
   - Sets up the main window and styling
   - Creates the tabbed interface
   - Initializes the face recognition system
   - Loads or trains models

2. **Interface Organization**:
   - Tab structure for different functionalities
   - Input areas for image upload or webcam capture
   - Output areas for recognition results and visualizations

3. **Recognition Process**:
   - User uploads an image or captures via webcam
   - User selects a recognition model (SVM, CNN, PCA+SVM)
   - System processes the image and displays the predicted person
   - Shows sample images of the predicted person from the dataset

4. **Performance Visualization**:
   - Displays accuracy metrics
   - Shows confusion matrices
   - For CNN, displays training/validation loss curves
   - For PCA+SVM, displays eigenfaces

5. **Dataset Exploration**:
   - Allows browsing through images of different people in the dataset

### Streamlit App Workflow

Similar to the Tkinter app but designed for web interface:

1. **Interface Layout**:
   - Sidebar for navigation and controls
   - Main area for content display

2. **Functionality**:
   - Model performance visualization
   - Face recognition from uploaded images
   - Dataset exploration

## Technical Details and Implementation

### Model Architectures

1. **SVM**:
   - Uses flattened pixel values as features
   - Applies standard scaling to normalize features
   - Uses RBF kernel for non-linear classification

2. **Neural Network (MLP)**:
   - Three hidden layers (512, 256, 128 neurons)
   - ReLU activation
   - Adam optimizer
   - Adaptive learning rate

3. **PCA+SVM**:
   - Reduces dimensionality to 50 principal components
   - Trains SVM on the reduced feature space
   - Visualizes eigenfaces (principal components)

4. **K-means**:
   - Unsupervised clustering with 40 clusters (same as the number of people)
   - Maps clusters to person IDs based on the most common true label in each cluster

### Image Preprocessing

1. For new images:
   - Convert to grayscale
   - Resize to 64x64 pixels (same as the dataset images)
   - Normalize pixel values to [0, 1]

2. For model input:
   - Flatten images to 1D arrays (4096 features)
   - Apply scaling or transformations as needed by each model

## Troubleshooting

Common issues and solutions:

1. **AttributeError: 'FaceRecognitionApp' object has no attribute 'selected_model'**
   - Cause: Variables are initialized after they're used in the app
   - Solution: Move the initialization of variables before the call to create_tabs() in the __init__() method

2. **ModuleNotFoundError**
   - Cause: Missing dependencies
   - Solution: Install all required packages with `pip install -r requirements.txt`

3. **Webcam not working**
   - Cause: OpenCV cannot access webcam
   - Solution: Ensure your webcam is connected and accessible, and you have proper permissions

4. **Slow model training**
   - Cause: Complex models take time to train
   - Solution: Reduce complexity (e.g., fewer components in PCA, fewer layers in NN)
